Battle for Wesnoth 1.12 Release Notes

December 2014. As the year comes to an end, a new stable release series begins. The development team is proud to present version 1.12 of The Battle for Wesnoth, a free and open source turn-based tactical strategy game with a high fantasy theme and role-playing elements. The game is available for Microsoft Windows, Apple OS X, and various GNU/Linux distributions.

What’s New in Wesnoth 1.12

For Players

This new stable release features plenty of interface improvements in all areas, starting with a refurnished game screen with an increased focus on units and terrains, and a more streamlined and modern visual design. As part of an increased focus in elegance and consistency, every dialog in the game benefits from the new design and many other subtle enhancements.

The minimap now offers alternative display modes for situations where a more fine-grained visualization of units, terrains, and villages is desired. Sidebar elements are rearranged and rescaled according to the current screen resolution in order to make better use of available space, and the team flag and number for the selected unit are displayed for ease of access.

Because Wesnoth thrives on the diversity and ingenuity of add-on content, this time the Add-ons Manager packs long overdue additions such as displaying each add-on’s installation status on the list, filtering items by their type or status, and enabling authors to link to feedback topics hosted on our official forum board.

Units with specialized variations like the Walking Corpse and Soulless have subsections in the built-in help system dedicated to documenting each variation in full. New top sections have been added as well, explaining the usage of add-ons for newcomers, and providing an overview for each multiplayer faction.

Many graphic improvements have been made for mounted units in the main game, including the Horseman, Cavalryman, and Elvish Scout unit lines; the Dwarvish Fighter and Thunderer lines; and the undead Spectre and Skeleton units. Several generic portraits were added or updated as well: the Spearman, Ogre, Wose, Ancient Lich, Wraith, and some troll units, amongst others.

Tropical forests were completely redrawn and expanded with several variations, including rainforest, sparse and dense palm forests, and savanna forests. Underground maps also benefit from the addition of new underground bridge types, and battle scenarios may be spiced up with two new trash and debris terrains available directly from the editor.

Campaigns

Many additional portrait variations have been added to Wesnoth’s flagship campaign, Heir to the Throne, as well as a brand new set of portrait art for Eastern Invasion.

All campaign journey maps featured in story screens — as well as the title screen — have been redrawn in a new style, and maps have been added to several campaigns that previously lacked one.

Most mainline campaigns have received visual and gameplay improvements on some level. Most notably, Son of the Black Eye has been rebalanced in its entirety, bringing its difficulty in line with its designation as an expert level campaign. Many scenarios of Under the Burning Suns have seen significant improvements, with some having been completely redesigned from the ground up.

General

Just as the game engine’s extensibility allows our community to create and publish their own original content, it also allows players to change options as they see fit. The Hotkeys section in the game Preferences dialog has been completely renewed with a categorized view and the ability to assign multiple sequences to a particular action, as well as change assignments to make use of additional buttons found in certain mouse models.

Very often, players require direct access to the game files to transfer or share with others their progress in campaigns, create or modify add-ons, and take screenshots of the game display or map terrain. As the game runs on multiple platforms and accepts a vast number of installation alternatives, a reliable and consistent mechanism to provide players with access to their stored data has been added to the Preferences dialog. In addition to this, pressing a screenshot key sequence configured in the Hotkeys section brings up a dialog with options to open the newly-created file with their preferred image viewer, or browse the full contents of their screenshots folder.

The UI theme selection dialog in Preferences/Display now includes a short description of each item. A new optional theme optimized for the Pandora and other small resolution devices has been added as well, providing a more compact visualization of unit stats and more visible command buttons. The WML and Lua API has received several changes which content creators porting 1.10-based add-ons must adapt to: a list is provided in this thread.

Both WML authors and players reporting bugs to them benefit from the greatly improved reports for loading-stage errors, displaying more legible error messages with the option to copy the contents to clipboard. The Gamestate Inspector tool provides the same option for game variables and status information; it now also uses the same underlying code as saved games to generate valid multiline WML attributes and translatable strings.

On the backend side, this new stable series contains a vast array of additions and changes for content creators. Micro AIs provide a powerful framework for assigning new and specialized AI behaviors to sides with minimal coding effort. Other AI improvements include a vastly improved and entirely configurable AI recruitment engine, and a new AI for use in multiplayer games.

As it has been the norm with previous stable series, many new features have been added to the WML and Lua API. For instance, animation WML tags have been extended to support a new, more compact syntax for defining animation frames in less space. Download

The Battle for Wesnoth is made available under the GNU General Public License (GPL). The source code is available from wesnoth.org, where you can also find instructions for building from source on a range of different operating systems. Ready-to-go packages are also available for most popular operating systems, including Microsoft Windows, Apple OS X, and GNU/Linux.
