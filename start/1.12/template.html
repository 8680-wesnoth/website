The Battle for Wesnoth 1.12

As the year comes to an end, a new stable release series begins.
The development team is proud to present version 1.12 of The Battle for Wesnoth!
This release features many improvements in all areas, including a completely revamped interface, a new multiplayer faction, dozens of new portraits, updated unit animations, hundreds of bug fixes, and a vastly improved map editor. Try out the new features for yourself, or read on for more details...

What’s New in Wesnoth 1.12

For Players

This new stable release features many interface improvements in all areas, starting with a refurnished game screen with an increased focus on units and terrains and a more streamlined and modern visual design. As part of the focus on elegance and consistency, every dialog in the game features the new design and many other subtle enhancements.

The minimap now offers alternative display modes for situations where a more fine-grained visualization of units, terrains, and villages is desired. Sidebar elements have been rearranged and rescaled according to the current screen resolution in order to make better use of available space, and the team flag and number for the selected unit are displayed for ease of access.

Because Wesnoth thrives on the diversity and ingenuity of add-on content, this time the Add-ons Manager packs long overdue additions, such as displaying each add-on’s installation status on the list, filtering items by their type or status, and enabling authors to link to feedback topics hosted on our official forum board.

Units with specialized variations like the Walking Corpse and Soulless have subsections in the built-in help system dedicated to documenting each variation in full. New top sections have been added as well, explaining the usage of add-ons for newcomers, and providing an overview for each multiplayer faction.

Many graphic improvements have been made for mounted units in the main game, including the Horseman, Cavalryman, and Elvish Scout unit lines. The Dwarvish Fighter and Thunderer lines and the undead Spectre and Skeleton units have had their animations revamped in full. Several generic portraits were added or updated as well: the Spearman, Ogre, Wose, Ancient Lich, Wraith, and some troll units, amongst others.

Campaigns

Many additional portrait variations have been added to Wesnoth’s flagship campaign, Heir to the Throne, as well as a brand new set of portrait art for Eastern Invasion.

All campaign journey maps featured in story screens — as well as the title screen — have been redrawn in a new style, and maps have been added to several campaigns that previously lacked one.

Most mainline campaigns have also received various visual and gameplay improvements. Most notably, Son of the Black Eye has been rebalanced in its entirety, bringing its difficulty in line with its designation as an expert level campaign. Many scenarios of Under the Burning Suns have seen significant improvements, with some having been completely redesigned from the ground up.

General

Just as the game engine’s extensibility allows our community to create and publish their own original content, it also allows players to change options as they see fit. Customizable Hotkeys were introduced and the Hotkeys Preferences dialog has been completely renewed with a categorized view. Players now have the ability to assign multiple sequences to a particular action, as well as change assignments to make use of additional buttons found in certain mouse models.

Very often, players require direct access to the game files to share their saves, create or modify add-ons, or take screenshots of the games or map creations. As the game runs on multiple platforms and accepts a vast number of installation alternatives, a reliable and consistent mechanism to provide players with access to their stored data has been added to the Preferences dialog. In addition to this, new options are now available to manage newly created screenshot: players will be able to immediately open them in their preferred image viewer, or browse the full contents of their screenshots folder on disk. The screenshot hotkey sequence has also been made customizable.

The UI theme selection dialog in Preferences/Display now includes a short description of each item. A new optional theme optimized for the Pandora and other small resolution devices has been added as well, providing a more compact visualization of unit stats and more visible command buttons. Two new translations have been started: Scottish Gaelic and Ukrainian. As always, The Battle for Wesnoth relies on volunteers for translating the game. Interested volunteers should refer to the documentation for translators on the Wesnoth.org wiki for details.

For Content Developers

The WML and Lua API has received several changes which content creators porting 1.10-based add-ons must adapt to: a list is provided in this thread.

Something something editor. [TODO: fabi]

Tropical forests were completely redrawn and expanded with several variations, including rainforest, sparse and dense palm forests, and savanna forests. Underground maps also benefit from the addition of new underground bridge types, and battle scenarios may be spiced up with two new trash and debris terrains available directly from the editor.

Both players and WML authors will benefit from improved debugging tools. Loading-stage error reports are now greatly more legible and feature copy-to-clipboard functionality. The Gamestate Inspector tool received similar updates: in addition the a copy-to-clipboard option for game variables and status information, it now also makes use of the same underlying code as saved games to correctly generate valid multiline WML attributes and translatable strings.

To further assist WML authors, a graphical interface has been added for the three main WML maintenance scripts (wmllint, wmlindent, and wmlscope), providing a convenient alternative to running them on the command line.

On the backend side, this new stable series contains a vast array of additions and changes for content creators. Micro AIs provide a powerful framework for assigning new and specialized AI behaviors to sides with minimal coding effort. Other AI improvements include a vastly improved and entirely configurable AI recruitment engine, and a new AI for use in multiplayer games.

As it has been the norm with previous stable series, many new features have been added to the WML and Lua API. For instance, animation WML tags have been extended to support a new, more compact syntax for defining animation frames in less space.
